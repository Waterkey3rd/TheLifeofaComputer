### 1. 整体技术栈与部署架构

本项目采用纯粹的无状态（Stateless）架构，满足高并发与轻量级化需求。

- 

  **前端 (Frontend)**：React + Tailwind CSS 。

- 

  **状态管理**：Zustand，用于处理复杂的 UI 状态与事件队列 。

- 

  **持久化存储**：LocalStorage，基于浏览器 UUID 标识独立存档，支持扫码即玩免注册 。

- 

  **后端 (Backend)**：FastAPI (Python)，纯无状态 API 设计 。

- 

  **数据校验**：Pydantic，严格约束接口数据与 JSON 事件模型 。

- 

  **部署方案**：Docker (docker-compose)，包含 Nginx 容器（前端分发与反向代理）和 Uvicorn 容器（后端 API） 。

### 4. 核心模块代码逻辑：事件调度引擎 (Dispatcher)

后端不依赖实时 API，而是作为高性能的发牌员 。

1. 

   **内存池与热更新**：启动时将 JSON 事件库加载到内存。通过暴露 `/api/admin/reload` 接口实现无需重启的无缝热更新 。

2. 

   **硬性过滤 (Hard Filter)**：对比前端传来的 `PlayerState`，剔除不满足 `prerequisites` 的事件 。

3. 

   **动态降权 (Dynamic Weighting)**：根据玩家近期遭遇的 `history_tags`，对相同标签的事件权重进行惩罚（如乘 0.2），保证极强的随机感 。

4. 

   **加权抽取与兜底**：使用加权随机算法抽取最终事件；若过滤池为空，返回极低权重的无条件“兜底事件”防止流程卡死 。

### 5. UI 描述与交互逻辑 (React + Tailwind CSS)

前端整体采用**“桌面操作系统 (OS) 隐喻”** ，使用 `100dvh` 完美适配移动端与桌面端 。

#### 5.1 页面级状态机映射

- 

  `BOOTING_VIEW`：加载页面。黑底绿字展示网协 Logo 闪烁，伴随开机自检音效 。

- 

  `SELECT_PC_VIEW`：新建存档。列表展示不同电脑机型，选定后注入初始属性 。

- 

  `DESKTOP_VIEW`：主游玩界面，包含核心操作逻辑 。

- 

  `GAME_OVER_VIEW`：结算界面。健康度归零触发“系统蓝屏”坏结局，存活天数达标触发好结局 。

#### 5.2 主视窗 (DESKTOP_VIEW) 交互布局

- 

  **左侧性能监控栏 (Stats Sidebar)**：类似任务管理器，用红/黄/绿动态进度条展示健康度和精神状态。移动端下优化为顶部横向滑动卡片以节省空间 。

- 

  **中央系统视窗 (Center Modal)**：所有日常或危机事件以“系统提示框”或对话框弹出。内部包含 `overflow-y-auto` 确保小屏设备上可滚动 。

  - 

    *限时选项动效*：按钮内置由蓝变红的倒退进度条组件，超时自动执行默认惩罚选项 。

- 

  **底部应用底座 (Dock Bar)**：使用绝对定位并在主区保留安全区（如 `pb-24`）防止重叠 。包含：【校园钱包】、【外设商店】、【校园服务预约】（网协及各维修店入口）、【我的背包】、【赛博百科】 。

- 

  **悬浮操作按钮 (FAB)**：独立的“结束今天”按钮放置在右下角。若存在未处理的事件视窗，该按钮被阻断禁用；点击后强制关闭其他应用弹窗并推进天数 。

#### 5.3 动画与视觉反馈

- 

  **受损反馈**：监听 Zustand 状态变化，扣除健康度时，触发全局 CSS `transform: translate` 震动特效及红色半透明滤镜 。

- 

  **增益反馈**：属性增加时，在对应进度条位置生成绿色的飘字组件，并播放清脆音效 。
