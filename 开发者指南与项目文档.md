# 赛博校园生存指南 (The Life of a Computer) - 项目文档与开发者指南

## 第一部分：项目设计与架构实现

### 1. 项目愿景与设计哲学
《赛博校园生存指南》是一款以“大学生电脑生存”为视角的网页端模拟策略游戏。玩家将扮演一台刚刚被带入校园的笔记本电脑，在充斥着流氓软件、意外断电、灰尘高温以及各种校园“作死”操作的环境下，努力存活 7 天（或在无尽模式中挑战极限）。
游戏旨在通过幽默、夸张的随机事件，向毫无计算机基础的大学生普及正确的电脑使用常识，并为“网协 (NPA)”等真实存在的校园技术服务组织进行宣传。

### 2. 技术栈实现
本项目采用前后端分离的现代化架构，确保了极高的扩展性与多端响应式体验：

- **前端层 (Frontend)**：
  - **核心框架**：React 18 + TypeScript + Vite。
  - **样式系统**：Tailwind CSS，实现了高度定制化的拟物/毛玻璃 UI，以及完善的移动端自适应（响应式 Flex/Grid 布局）与暗黑模式 (Dark Mode)。
  - **状态管理**：Zustand，并启用了 `persist` 中间件。游戏存档（UUID、进度、硬件健康、背包、标志位）会被全自动、实时地序列化保存在浏览器的 LocalStorage 中，实现**纯前端免注册存档**。
  - **页面路由**：使用轻量级的组件渲染级路由 (`BootingView` -> `SelectPCView` -> `DesktopView` -> `GameOverView`)。

- **后端层 (Backend)**：
  - **核心框架**：Python 3 + FastAPI。
  - **数据验证**：Pydantic V2 构建了极其严谨的数据模型 (`PlayerState`, `EventSchema`)。
  - **逻辑驱动**：无状态 (Stateless) 的 API 设计。前端在请求事件时将当前的 `PlayerState` 以 JSON 发送给后端，后端通过 `logic_engine.py` 计算并应用事件修正后，返回新的状态给前端，极大地减轻了服务端的并发内存压力。
  - **事件分发**：`dispatcher.py` 会从文件系统中读取 JSON 事件池，根据前置条件（常识要求、天数限制、资产等）和动态权重进行事件随机抽取。

## 第二部分：开发者更新与维护指南

作为此项目的维护者，你需要掌握如何添加和修改游戏内容。以下是详尽的示例指南。

### 1. 如何添加新的“事件” (核心玩法)

事件系统采用 JSON 格式驱动。你可以通过新建 JSON 文件或者使用 LLM 脚本生成来快速扩充。

#### 1.1 参数字典 (写事件必备)
在编写 `stat_changes`（属性变化）或 `prerequisites`（触发条件）时，**只能使用以下固定的键名 (Key)**：

**状态变化属性池 (`stat_changes` 可用的 Key)：**
| 键名 (Key) | 描述 |
| :--- | :--- |
| `hw_cpu` | CPU 健康度 (扣除写负数) |
| `hw_disk` | 硬盘 健康度 |
| `hw_ram` | 内存 健康度 |
| `hw_screen` | 屏幕 健康度 |
| `hw_fan` | 风扇 健康度 |
| `hw_shell` | 外壳 健康度 |
| `health_system` | 系统稳定性 |
| `health_storage` | 存储空间利用率 |
| `health_software` | 软件纯净度 |
| `mental_state` | 玩家精神状态 |
| `wealth` | 玩家金钱 (人民币) |
| `cyber_sense` | 赛博常识点数 |

**前置条件判断池 (`prerequisites` 可用的 Key)：**
| 键名 (Key) | 类型 | 示例用法 | 描述 |
| :--- | :--- | :--- | :--- |
| `min_day` | 整数 | `{"min_day": 3}` | 第几天之后才可能触发此事件 |
| `max_day` | 整数 | `{"max_day": 5}` | 第几天之前才可能触发此事件 |
| `min_wealth` | 整数 | `{"min_wealth": 500}` | 金钱大于该值触发 |
| `min_cyber_sense`| 整数 | `{"min_cyber_sense": 10}` | 常识点数大于该值触发 |
| `required_item` | 字符串 | `{"required_item": "外星人散热架"}` | 背包里必须有该道具 |

#### 1.2 完整的 JSON 事件书写示例
创建一个 `backend/data/events/routine/coffee_spill.json`：
```json
{
  "event_id": "evt_coffee_spill_01",
  "event_type": "routine",
  "tags": ["hardware_danger", "liquid"],
  "is_unique": true,
  "weight": 20,
  "prerequisites": {
    "min_day": 2
  },
  "title": "咖啡杯的背叛",
  "description": "你的室友在打游戏时过于激动，一巴掌碰倒了桌上的美式咖啡。褐色的液体正顺着桌子流向你的键盘缝隙！",
  "technical_context": "液体会导致主板短路，此时最危险的行为就是保持通电。",
  "timeout_seconds": 5,
  "timeout_option_id": "opt_panic",
  "options": [
    {
      "option_id": "opt_force_shutdown",
      "text": "立刻长按电源键强制关机，并拔掉电源！",
      "required_cyber_sense": 10,
      "outcomes": [
        {
          "probability": 1.0,
          "result_text": "你果断的急救保住了主板！只有几个按键变得黏糊糊的。",
          "stat_changes": {
            "hw_shell": -5,
            "mental_state": -10
          }
        }
      ]
    },
    {
      "option_id": "opt_panic",
      "text": "大声尖叫，拿抽纸在表面疯狂擦拭！",
      "required_cyber_sense": 0,
      "outcomes": [
        {
          "probability": 0.8,
          "result_text": "在擦拭的过程中咖啡渗入了主板，你闻到了一股刺鼻的焦糊味。电脑瞬间黑屏了。",
          "stat_changes": {
            "hw_cpu": -30,
            "hw_ram": -20,
            "wealth": -200
          }
        },
        {
          "probability": 0.2,
          "result_text": "奇迹发生了，你的外壳防水性能不错，没有渗入主板。",
          "stat_changes": {
            "hw_shell": -10
          }
        }
      ]
    }
  ]
}
```

> **提示：** 事件如果含有 `timeout_seconds` 参数，会在前端展示紧急进度条，如果倒计时结束玩家未操作，系统会自动选择 `timeout_option_id` 对应的选项。

#### 1.3 使用大模型一键生成 (推荐)
- 运行：`python generateEvent_tool.py`
- 脚本内已内置了严谨的 Prompt 与上述字典。你只需输入“室友拔插头”或“下载流氓软件”，AI 将自动为你生成多分支、带概率的完美 JSON 事件代码。

### 2. 如何添加新的“初始电脑类型”
- **文件路径**：`frontend/src/views/SelectPCView.tsx`
- **操作步骤**：
  在 `availablePCs` 数组中插入新卡片：
```typescript
{
  model_id: 'rog_strix_2024',
  display_name: 'ROG 枪神 2024',
  base_health_hardware: 110,  // 初始硬件强度
  base_health_system: 95,     // 初始系统稳定性
  cooling_material: 'liquid_metal', // 散热材质：silicone(硅脂) 或 liquid_metal(液金)
  is_memory_upgradeable: true,      // 是否支持扩展内存
  is_apple_silicon: false,          // 是否为 ARM 架构
  price_tier: 2.5,                  // 维修费用的倍率
}
```

### 3. 如何添加新的“商店商品” (外设商店)
- **文件路径**：`frontend/src/components/Apps.tsx`
- **操作步骤**：
  在 `StoreApp` 组件的 `officialItems` 或 `xianyuItems` 数组中添加商品：
```typescript
{ 
  id: 'item_ups', 
  name: '小型 UPS 不间断电源', 
  price: 299, 
  desc: '防止宿舍突然断电导致硬盘报废。', 
  // target目前用于UI归类，数值暂未深度耦合，通常设为 0
  effects: { target: 'disk', val: 0 } 
}
```

### 4. 如何添加新的“百科文章” (网协知识库)
- **文件路径**：`frontend/src/components/Apps.tsx`
- **操作步骤**：
  在 `WikiApp` 的 `articles` 数组中添加：
```typescript
{
  id: 'wiki_water',
  title: '电脑进水急救指南',
  content: '千万别用吹风机热风吹！这会把水珠吹进主板深处。正确的做法是：立刻断电（拔掉电池栏）、长按电源键强制关机，然后倒置在阴凉通风处。',
  unlockDay: 2, // 游戏到第 2 天才会解锁此文章
  gain: 15      // 首次阅读后给予玩家的“赛博常识”奖励
}
```

## 第三部分：部署与服务管理

### 开发环境联调
启动前端：
```bash
cd frontend
npm run dev
```
启动后端 (务必使用内置脚本以绑定 0.0.0.0，否则局域网手机无法访问)：
```bash
cd backend
python run.py
```

### 存档与测试排错
由于采用了纯前端的 Zustand 持久化存储：
- **存档机制**：数据保存在浏览器 `localStorage` 中（键名为 `cyber-survival-save`）。
- **清空重置**：如果修改了数据模型结构导致前后端不一致报错（红字网络错误），可以在浏览器按下 F12 进入控制台执行：
  `localStorage.removeItem('cyber-survival-save')` 并刷新页面。
- **无尽模式**：在选电脑界面勾选“无尽模式”后，将取消 7 天强制通关的限制，支持在主界面通过点击系统监控栏处的**红色电源按钮长按1秒**随时退出至菜单，并在菜单点击“唤醒系统”以中途切换模式。
